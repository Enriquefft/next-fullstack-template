## Accessibility

**Checklist**:
- [ ] Keyboard navigation works on all interactive elements
- [ ] Focus states are visible (using `--ring` CSS variable)
- [ ] Screen reader labels on icon buttons (`aria-label`)
- [ ] Color contrast meets WCAG AA standard (4.5:1 minimum)
- [ ] Touch targets are â‰¥ 44px on mobile
- [ ] Semantic HTML elements used (`<button>`, `<nav>`, `<main>`, etc.)
- [ ] Form inputs have associated `<label>` elements
- [ ] Error messages are announced to screen readers
- [ ] Loading states are communicated (`aria-busy`, `aria-live`)

**ARIA Attributes**:
- `aria-label`: Descriptive labels for icon-only buttons
- `aria-labelledby`: Associate headings with sections
- `aria-describedby`: Associate descriptions with form fields
- `aria-busy`: Indicate loading states
- `aria-live="polite"`: Announce dynamic content changes
- `role="status"`: Status messages
- `role="alert"`: Important, time-sensitive messages
